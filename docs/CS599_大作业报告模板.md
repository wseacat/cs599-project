## 一、选题背景与设计思想

### 1.1 问题定义

企业内部存在大量文档（制度手册、技术文档、产品说明等），员工在查找信息时面临以下痛点：

1. **检索效率低**：传统关键词搜索难以理解语义
2. **信息分散**：答案可能分布在多个文档中
3. **缺乏上下文**：无法进行多轮对话深入探讨
4. **来源不明确**：无法验证答案的准确性

### 1.2 现有方案不足

传统 RAG 系统（retrieve → answer）存在以下问题：

1. **单次检索**：无法处理复杂问题
2. **无质量判断**：检索结果不佳时无法自适应
3. **缺乏规划**：无法拆解复杂问题
4. **无引用溯源**：答案可信度低

### 1.3 项目价值

本项目构建的 Agentic-RAG 系统具有以下价值：

1. **智能检索**：多阶段 Agent 工作流，自动优化检索
2. **自适应**：Reflection 机制判断检索质量，智能重试
3. **引用溯源**：每个答案都有明确的文档来源
4. **多轮对话**：支持上下文关联的连续问答

### 1.4 技术路线

```
用户提问
    ↓
Planner Agent (问题分析)
    ↓
Query Agent (Query Rewrite/Expansion)
    ↓
Retriever Agent (Hybrid Retrieval)
    ↓
Rerank Agent (结果重排序)
    ↓
Reflection Agent (质量判断)
    ├─ pass → Answer Agent (生成答案)
    └─ fail → 重试检索
```

### 4.1 LangGraph Workflow 定义

```python
# backend/src/workflows/rag_workflow.py

workflow = StateGraph(RAGState)

# 添加节点
workflow.add_node("planner", planner_agent)
workflow.add_node("query", query_agent)
workflow.add_node("retriever", retriever_agent)
workflow.add_node("rerank", rerank_agent)
workflow.add_node("reflection", reflection_agent)
workflow.add_node("answer", answer_agent)

# 定义边
workflow.add_edge("planner", "query")
workflow.add_edge("query", "retriever")
workflow.add_edge("retriever", "rerank")
workflow.add_edge("rerank", "reflection")

# 条件边：Reflection 判断是否重试
workflow.add_conditional_edges(
    "reflection",
    should_retry,
    {
        "retry": "query",
        "pass": "answer"
    }
)

workflow.add_edge("answer", END)
```

### 4.2 Agent 实现示例

```python
# backend/src/agents/planner.py

async def planner_agent(state: RAGState) -> Dict:
    """分析问题，拆解检索任务"""
    
    query = state["original_query"]
    chat_history = state.get("chat_history", [])
    
    # 调用 LLM 分析问题
    prompt = f"""分析以下问题，判断是否需要拆解：
    
    问题：{query}
    对话历史：{chat_history}
    
    请判断：
    1. 问题是否复杂
    2. 是否需要拆解为多个子问题
    3. 生成检索计划
    """
    
    response = await llm.ainvoke(prompt)
    
    return {
        "is_complex": response.is_complex,
        "sub_queries": response.sub_queries,
        "retrieval_plan": response.retrieval_plan
    }
```

### 4.3 混合检索实现

```python
# backend/src/retrieval/hybrid.py

class HybridRetriever:
    """融合 Vector 和 BM25 检索"""
    
    async def retrieve(self, query: str, top_k: int = 10):
        # 并行执行两种检索
        vector_results, bm25_results = await asyncio.gather(
            self.vector_search(query, top_k),
            self.bm25_search(query, top_k)
        )
        
        # 融合结果 (RRF - Reciprocal Rank Fusion)
        fused_results = self.reciprocal_rank_fusion(
            vector_results, 
            bm25_results
        )
        
        return fused_results[:top_k]
```
