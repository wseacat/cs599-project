## 一、选题背景与设计思想

### 1.1 问题定义

企业内部存在大量文档（制度手册、技术文档、产品说明等），员工在查找信息时面临以下痛点：

1. **检索效率低**：传统关键词搜索难以理解语义，用户需要精确匹配关键词才能找到信息
2. **信息分散**：一个问题的答案可能分布在多个文档中，需要人工整合
3. **缺乏上下文**：无法进行多轮对话深入探讨问题
4. **来源不明确**：无法验证答案的准确性，缺乏可信度

### 1.2 现有方案不足

传统 RAG 系统采用简单的 `retrieve → answer` 流程，存在以下问题：

| 问题 | 表现 | 影响 |
|------|------|------|
| 单次检索 | 只执行一次检索，无法处理复杂问题 | 答案不完整 |
| 无质量判断 | 检索结果不佳时无法自适应 | 答案质量差 |
| 缺乏规划 | 无法拆解复杂问题 | 无法处理复合查询 |
| 无引用溯源 | 答案缺乏来源标注 | 可信度低 |

### 1.3 项目价值

本项目构建的 Agentic-RAG 系统具有以下价值：

1. **智能检索**：多阶段 Agent 工作流，自动优化检索策略
2. **自适应**：Reflection 机制判断检索质量，智能重试
3. **引用溯源**：每个答案都有明确的文档来源，可验证
4. **多轮对话**：支持上下文关联的连续问答

### 1.4 技术路线

本项目采用 Agentic-RAG 架构，核心思想是将传统 RAG 的单一检索流程改造为多 Agent 协作的智能工作流：

```
传统 RAG:  User Query → Retrieve → Answer

Agentic-RAG:
User Query
    ↓
Planner Agent → 分析问题，拆解检索任务
    ↓
Query Agent → Query Rewrite / Expansion
    ↓
Retriever Agent → Hybrid Retrieval (Vector + BM25)
    ↓
Rerank Agent → BGE Reranker 重排序
    ↓
Reflection Agent → 判断检索质量
    ├─ pass → Answer Agent → 生成答案 + 引用来源
    └─ fail → 重试检索 (max 2次)
```

### 1.5 核心技术要素

本项目涵盖以下核心技术要素：

| 技术要素 | 实现方式 |
|----------|----------|
| SDD 规格驱动开发 | Product Spec / Architecture Spec / API Spec |
| 工具使用 / Function Calling | LangGraph Agent 节点调用检索、重排序等工具 |
| 记忆机制 | Redis 存储对话历史，支持多轮上下文 |
| 状态管理与多步骤推理 | LangGraph StateGraph 管理工作流状态 |
| 多智能体协作 | 6 个 Agent 分工协作完成检索问答 |
| 可观测性 | 结构化日志 + Retrieval Debug 可视化 |

### 3.2 Agent 交互流程

系统包含 6 个 Agent，各司其职：

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| Planner | 分析问题，拆解任务 | User Query | Retrieval Plan |
| Query | Query Rewrite/Expansion | Original Query | Rewritten/Expanded Queries |
| Retriever | 混合检索 | Queries | Retrieved Documents |
| Rerank | 重排序 | Documents | Reranked Documents |
| Reflection | 质量判断 | Documents | Pass/Fail Decision |
| Answer | 生成答案 | Documents + Query | Answer + Citations |

### 3.3 数据流设计

#### 文档处理流程

```
文档上传 → 文档解析 → 文本切分 → Embedding 生成 → 存入 Milvus + BM25 索引
```

- **文档解析**：支持 PDF (PyMuPDF)、DOCX (python-docx)、TXT、Markdown
- **文本切分**：RecursiveCharacterTextSplitter，chunk_size=500，overlap=100
- **Embedding**：BGE Large Zh (bge-large-zh-v1.5)，1024 维

#### 问答流程

```
用户提问 → Planner 拆解 → Query 优化 → 混合检索 → Rerank → Reflection → Answer
```

- **混合检索**：Vector Search (Milvus) + BM25，RRF 融合排序
- **重排序**：BGE Reranker (bge-reranker-large)
- **自省**：判断检索质量，不满意则重试（最多 2 次）

### 3.4 技术选型理由

| 组件 | 选型 | 理由 |
|------|------|------|
| Agent 框架 | LangGraph | 原生支持状态机、条件分支、循环 |
| 向量数据库 | Milvus | 高性能、分布式、生产级 |
| 缓存 | Redis | 高性能、支持多种数据结构 |
| Embedding | BGE Large Zh | 中文效果最佳，MTEB 排名领先 |
| Reranker | BGE Reranker | 与 Embedding 配套，效果好 |
| 后端框架 | FastAPI | 异步、自动文档、类型安全 |
| 前端框架 | Vue3 | 响应式、组合式 API、生态成熟 |

# backend/src/workflows/rag_workflow.py

def build_rag_workflow() -> StateGraph:
    """Build the RAG workflow graph with all agents."""
    graph = StateGraph(RAGState)

    # Add agent nodes
    graph.add_node("planner", planner_agent)
    graph.add_node("query_agent", query_agent)
    graph.add_node("retriever", retriever_agent)
    graph.add_node("rerank", rerank_agent)
    graph.add_node("reflection", reflection_agent)
    graph.add_node("answer", answer_agent)

    # Define workflow edges
    graph.set_entry_point("planner")
    graph.add_edge("planner", "query_agent")
    graph.add_edge("query_agent", "retriever")
    graph.add_edge("retriever", "rerank")
    graph.add_edge("rerank", "reflection")

    # Conditional edge: retry or answer
    graph.add_conditional_edges(
        "reflection",
        should_retry,
        {"answer": "answer", "query_agent": "query_agent"}
    )

    graph.add_edge("answer", END)
    return graph.compile()
```

# backend/src/agents/planner.py

async def planner_agent(state: RAGState) -> dict:
    """Analyze user query and generate retrieval plan."""
    query = state["original_query"]
    chat_history = state.get("chat_history", [])

    # Build conversation history context
    history_text = ""
    if chat_history:
        history_text = "\n".join([f"{msg.type}: {msg.content}" for msg in chat_history[-6:]])

    llm = get_llm()
    prompt = f"""你是一个查询分析专家。分析用户问题并生成检索计划。

用户问题：{query}

{f'对话历史：{history_text}' if history_text else ''}

请返回JSON格式的检索计划：
{{
    "needs_decomposition": true/false,
    "sub_queries": ["子查询1", "子查询2"],
    "strategy": "描述检索策略",
    "key_entities": ["关键实体1", "关键实体2"]
}}

只返回JSON，不要其他内容。"""

    try:
        response = await llm.ainvoke(prompt)
        plan_text = _extract_text(response.content)
        plan = json.loads(plan_text)
    except Exception as e:
        logger.warning("planner_parse_failed", error=str(e))
        plan = {"needs_decomposition": False, "sub_queries": [query]}

    return {
        "retrieval_plan": json.dumps(plan, ensure_ascii=False),
        "workflow_trace": state.get("workflow_trace", []) + [{"step": "planner", "output": plan}],
    }
```

### 4.4 混合检索实现

```python
# backend/src/retrieval/hybrid.py

async def hybrid_search(query: str, top_k: int = 10) -> list[dict]:
    """融合 Vector 和 BM25 检索"""
    # Check cache first
    cache_key = f"search:{hashlib.md5(query.encode()).hexdigest()}:{top_k}"
    try:
        redis = await get_redis()
        cached = await redis.get(cache_key)
        if cached:
            return json.loads(cached)
    except Exception:
        pass

    # Run BM25 and vector search in parallel
    bm25_index = get_bm25_index()
    bm25_task = asyncio.to_thread(bm25_index.search, query, top_k)
    vector_task = search_similar(query, top_k=top_k)

    bm25_results, vector_results = await asyncio.gather(
        bm25_task, vector_task, return_exceptions=True,
    )

    # Merge by chunk_id
    merged: dict[int, dict] = {}
    for r in bm25_results:
        key = r["chunk_id"]
        merged[key] = {**r, "bm25_score": r["score"], "vector_score": 0.0}

    for r in vector_results:
        key = r["chunk_id"]
        if key in merged:
            merged[key]["vector_score"] = r["score"]
        else:
            merged[key] = {**r, "bm25_score": 0.0, "vector_score": r["score"]}

    # RRF ranking
    ranked = rrf_rank(list(merged.values()), top_k=top_k)

    # Cache results
    try:
        redis = await get_redis()
        await redis.setex(cache_key, 300, json.dumps(ranked, ensure_ascii=False))
    except Exception:
        pass

    return ranked


def rrf_rank(results: list[dict], k: int = 60, top_k: int = 10) -> list[dict]:
    """Reciprocal Rank Fusion"""
    for r in results:
        bm25_rank = r.get("bm25_rank", k + 1)
        vector_rank = r.get("vector_rank", k + 1)
        r["rrf_score"] = 1.0 / (k + bm25_rank) + 1.0 / (k + vector_rank)

    return sorted(results, key=lambda x: x["rrf_score"], reverse=True)[:top_k]
```

### 4.5 SSE 流式输出实现

```python
# backend/src/api/v1/chat.py

@router.post("/stream")
async def chat_stream_endpoint(
    request: ChatRequest,
    current_user: User = Depends(get_current_user),
):
    """SSE streaming with real-time agent progress events."""

    async def event_generator():
        try:
            async for event_type, event_data in chat_stream(
                user_id=current_user.id,
                message=request.message,
                conversation_id=request.conversation_id,
            ):
                if event_type == "progress":
                    yield {
                        "event": "progress",
                        "data": json.dumps(event_data, ensure_ascii=False),
                    }
                elif event_type == "final":
                    yield {
                        "event": "token",
                        "data": json.dumps({"text": event_data.get("answer", "")}, ensure_ascii=False),
                    }
                    yield {
                        "event": "final",
                        "data": json.dumps(event_data, ensure_ascii=False, default=str),
                    }
                elif event_type == "error":
                    yield {
                        "event": "error",
                        "data": json.dumps(event_data, ensure_ascii=False),
                    }
        except Exception as e:
            yield {"event": "error", "data": json.dumps({"detail": str(e)})}

    return EventSourceResponse(event_generator())
```

## 七、课程总结

### 7.1 个人收获

1. **工程思维转变**：从"代码编写者"到"智能体编排者"的转变。学会了如何设计 Agent 工作流，而不是简单地写函数调用。

2. **SDD 方法论**：实践了规格驱动开发，先写 Spec 再写代码，提高了开发效率和代码质量。

3. **系统设计能力**：学会了如何设计可扩展、可维护的系统架构，理解了分层架构和依赖注入的价值。

4. **AI 工程化**：理解了 AI 应用从原型到生产的完整链路，包括错误处理、性能优化、可观测性等。

### 7.2 对课程的建议

1. 增加 Agent 评估的实践指导，如何科学地评估 Agent 系统的效果

本项目从零构建了一个企业级 Agentic-RAG 智能知识库系统，实现了多阶段检索、自适应优化和精准引用。通过这个项目，我深入理解了 Agentic AI 的设计理念和工程实践，为未来的 AI 应用开发打下了坚实的基础。
